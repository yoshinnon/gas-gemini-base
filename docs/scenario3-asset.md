# シナリオ3: 多通貨ポートフォリオ管理

## 概要

**目的:** 銀行明細・証券口座・暗号資産ウォレットなど、バラバラな金融テキストを AI が解析し、すべて日本円（JPY）換算で資産を一元管理する。

**用途:**
- 複数の金融機関の残高をまとめて把握したい
- 暗号資産の JPY 換算額をリアルタイムに近い値で確認したい
- 過去の資産合計推移をグラフで可視化したい
- どんな形式のテキストでも（CSV・スクリーンコピー等）対応したい

> ⚠️ パスワード・秘密鍵など認証情報は絶対に入力しないでください。残高・評価額のみを貼り付けてください。

## データフロー

```

（金融テキストを貼り付け）
        ↓ メニュー「💰 資産データ解析・更新」
presentation/AssetMenu.ts
        ↓
core/AssetUseCase.ts
   ├── core/AssetParser.ts     ← AI が資産・通貨を抽出
   └── infra/FinanceApi.ts     ← レート取得（3段フォールバック）
        ↓ JPY 換算
gas/AssetRepository.ts
   ├── Portfolio シート（差分更新）
   └── History シート（追記・グラフ更新）
```

## 為替レート取得の3段フォールバック

| 優先度 | API | 対象通貨 | 無料枠 | 備考 |
|--------|-----|---------|--------|------|
| 1 | [ExchangeRate-API](https://open.er-api.com/v6/latest/JPY) | 法定通貨（USD, EUR 等） | 1,500 req/月 | 登録不要 |
| 2 | [CoinGecko API](https://api.coingecko.com/) | 暗号資産（BTC, ETH 等） | 10,000 req/月 | 登録不要 |
| 3 | Gemini AI 推論 | 上記 API が失敗した場合 | Gemini 無料枠内 | ⚠️ 推定値 |

AI 推論レートを使用した行には「⚠️ 推定」フラグが付与されます。投資判断には公式レートをご確認ください。

## Portfolio シートの列構成

| 列 | 内容 | 更新方針 |
|----|------|---------|
| identifier | 差分更新キー（非表示） | 内部管理 |
| カテゴリー | 資産クラス | 自動更新 |
| 資産名 | 資産の正式名称 | 自動更新 |
| 数量 | 残高・保有数量 | 自動更新 |
| 通貨 | ISO 4217 コード等 | 自動更新 |
| レート(JPY) | 1単位あたりの JPY レート | 自動更新 |
| JPY換算額 | 数量 × レート | 自動更新 |
| 取得モデル | 使用した Gemini モデル | 自動記録 |
| レートソース | exchangerate-api / coingecko / ai-estimate | 自動記録 |
| 推定フラグ | 実データ / ⚠️ 推定 | 自動記録 |
| 最終更新 | 更新日時 | 自動更新 |
| **ステータス** | **保有中 / 売却済み / 確認中 / 除外** | **ユーザー編集（保護）** |
| **メモ** | **自由記述** | **ユーザー編集（保護）** |

太字の列（ステータス・メモ）は再実行時でも**絶対に上書きされません**。

---

## 差分更新ロジック

再実行するたびに全データを消して書き直すのではなく、`identifier`（資産の一意キー）で照合します:

- **既存資産（identifier 一致）:** 数量・レート・JPY換算額・最終更新のみ更新
- **新規資産（identifier なし）:** 末尾に新しい行を追加
- **削除:** 手動で行を削除してください（自動削除はしません）

---

## History シートと資産推移グラフ

実行のたびに以下が自動記録されます:

| 列 | 内容 |
|----|------|
| 記録日時 | 実行日時 |
| JPY合計 | 全資産の JPY 換算合計 |
| 資産件数 | Portfolio シートの件数 |
| 使用モデル | Gemini モデル名 |
| AI推定レート有無 | あり / なし |
| 備考 | 入力データのサマリー |

History シートには資産合計のラインチャートが自動生成されます（実行のたびに更新）。

## 汎用性：対応する入力形式の例

このパーサーは特定の金融機関に依存しません。以下のような様々な形式に対応します:

**例1: 銀行アプリのスクリーンコピー**
```
三菱UFJ銀行
普通預金　1,234,567円
定期預金　500,000円
```

**例2: 証券口座の評価損益画面**
```
トヨタ自動車 (7203)
保有数量: 100株
現在値: 3,250円
評価額: 325,000円
```

**例3: 暗号資産ウォレット**
```
Bitcoin: 0.05234 BTC
Ethereum: 1.234 ETH
USDT: 500.00
```

**例4: 混在テキスト（広告・UI 要素が混入していても可）**
```
口座残高照会
ログイン　パスワードを入力してください  ← 無視される
普通預金残高: ¥450,000
定期預金残高: ¥1,000,000
広告バナー: お得なキャンペーン実施中！  ← 無視される
```

## テスト

```bash
npx jest tests/core/AssetParser.test.ts
npx jest tests/infra/FinanceApi.test.ts
```

**テストカバレッジ:**
- 正常な JSON パース（複数資産）
- Markdown コードブロック除去
- 無効なカテゴリー → Other フォールバック
- 無効な通貨コード → UNKNOWN フォールバック
- 負の amount → 0 に補正
- identifier の重複解消（_2 サフィックス）
- JPY 換算計算（rate=0 → null）
- 通貨コードの抽出（重複除去）
