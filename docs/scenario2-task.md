# シナリオ2: AI 議事録解析・タスク自動抽出

## 概要

**目的:** 会議メモ・議事録のテキストから、参加者が「やること」として認識しているアクションアイテムを AI が自動抽出し、スプレッドシートのタスクボードに整理する。

**用途:**
- 会議後にメモをコピペするだけでタスク管理ができる
- 「来週月曜」などの相対日付を自動で具体的な日付に変換したい
- 担当者・優先度・期限を一覧で把握したい
- 議事録の文体（敬語・曖昧表現）を正しく解釈させたい

---

## データフロー

```
Input シート
（会議メモを貼り付け）
       ↓ メニュー「📝 AI議事録解析を実行」
presentation/TaskMenu.ts
       ↓ 確認ダイアログ → 処理開始
core/TaskUseCase.ts
       ↓ AI API 呼び出し（リトライ付き）
core/TaskExtractor.ts
   ├── buildSystemPrompt()  ← JSON スキーマ強制
   ├── buildUserPrompt()    ← 議事録テキスト
   └── parseTaskResponse()  ← AI 出力をパース
       ↓ JSON パース成功
gas/TaskSheetAdapter.ts
       ↓ TaskBoard シートへ追記
TaskBoard シート
（タスク一覧・ドロップダウン・色分け）
```

## スプレッドシートの準備

| シート名 | 作成方法 | 用途 |
|---------|---------|------|
| `Input` | **手動作成が必要** | 議事録・会議メモを貼り付ける |
| `TaskBoard` | 初回実行時に自動作成 | タスク一覧 |
| `AI_Log` | 初回実行時に自動作成 | 実行ログ |

**Input シートの作り方:**
スプレッドシート下部の「+」ボタンでシートを追加し、シート名を `Input` に変更する。

## TaskBoard シートの列構成

| 列 | 内容 | 自動設定 |
|----|------|---------|
| ステータス | 未着手 / 進行中 / 完了 / 保留 | ドロップダウン |
| 期限 | YYYY-MM-DD 形式 | 相対日付を自動変換 |
| 担当者 | 抽出された担当者名 | — |
| 優先度 | HIGH / MEDIUM / LOW | 行背景色で視覚化 |
| タイトル | タスクの簡潔なタイトル | — |
| 内容 | 詳細説明・完了条件 | — |
| 元モデル | 抽出に使用した Gemini モデル名 | 自動記録 |
| 登録日時 | 抽出実行日時 | 自動記録 |

**優先度の行背景色:**

| 優先度 | 背景色 |
|--------|--------|
| HIGH | 薄い赤（#fce8e6） |
| MEDIUM | 薄い黄（#fef9e3） |
| LOW | 薄い緑（#e6f4ea） |

## プロンプト設計のポイント

### 日本語特有の表現への対応

`buildSystemPrompt()` では以下のルールを AI に指示しています:

**タスクとして抽出する表現:**
- 「〜してください」「〜をお願いします」
- 「〜することになりました」「〜を確認します」
- 「TODO」「宿題」「持ち帰り」

**タスクとして抽出しない表現:**
- 「〜かもしれません」（意思が不確定）
- 「〜という話もありました」（議題のみ、アクションなし）

### 相対日付の変換

実行日をプロンプトに埋め込み、AI に具体的な日付への変換を指示します:

| 表現 | 変換方法 |
|------|---------|
| 「来週月曜」 | 実行日の翌週月曜日 |
| 「今週中」 | 実行日の週の金曜日 |
| 「月末」 | 実行日の月の末日 |
| 「3日以内」 | 実行日 + 3日 |
| 「本日」「今日中」 | 実行日そのもの |
| 期限の記載なし | 空文字 |

## リトライ設計

JSON パース失敗時に最大2回リトライします（モデルフォールバックとは独立）:

```
1回目: 通常のプロンプト
    ↓ JSON パース失敗
2回目: 「前回の出力は有効な JSONではありませんでした」のフィードバック付き
    ↓ JSON パース失敗
3回目: 同上
    ↓ JSON パース失敗
→ Error をスロー（「リトライ上限を超えました」）
```

---

## エラーハンドリング仕様

| 状況 | ユーザーへの表示 |
|------|----------------|
| Input シートが存在しない | エラーダイアログ（シート作成を促す） |
| Input シートが空 | エラーダイアログ（テキスト貼り付けを促す） |
| APIキー未設定 | 設定ガイドへ誘導 |
| 429（レートリミット）| 自動フォールバック（ユーザーには見えない） |
| 全モデル失敗 | エラーダイアログ（待機してから再実行を促す） |
| JSON パース失敗（リトライ超過）| エラーダイアログ |
| タスク0件 | 完了ダイアログ（0件の旨を表示） |

## 使い方

1. スプレッドシートに `Input` シートを作成する
2. `Input` シートに会議メモをそのまま貼り付ける（形式は問わない）
3. メニュー「📝 AI議事録解析を実行」をクリック
4. 確認ダイアログで「はい」を選択
5. 処理完了後、`TaskBoard` シートにタスクが追加される

**Input シートの例:**
```
2025年7月1日 週次定例ミーティング

参加者: 田中、鈴木、山田

[議題1] API設計
→ 田中さんにREST APIの設計書をConfluenceに作成してもらう（来週月曜まで）

[議題2] テスト環境
→ 鈴木さんがステージング環境の構築を今週中に対応する予定

[議題3] 次回の日程
→ 山田さんが全員の空き時間を確認して日程調整する
```

**テストカバレッジ:**
- 正常なJSONパース
- Markdownコードブロック除去
- 不正JSONでのエラー
- 優先度バリデーション（不正値→MEDIUM）
- dueDate バリデーション（非ISO形式→空文字）
- フォールバック発生時の挙動
- JSONパース失敗リトライ
- 全モデル失敗時の即時エラー
